Tech Notes

2D Isometric by Design (Not True 3D)
Outputs are 2D images in a strict isometric projection, not 3D meshes. This keeps the system fast, lightweight, and workflow-friendly.
Rule-Constrained Angles (Not Prompt Guessing)
Angle consistency is enforced with explicit geometric constraints to avoid perspective drift and mismatched views.
Multi-Output Selection
Each run generates multiple candidates so you can pick the most accurate angle/structure, reducing "one-shot randomness."
Material Preservation
With clear inputs, textures and materials are preserved with high fidelity, prioritizing reconstruction over stylization.
Stability Roadmap
The demo uses a shared core algorithm; the full version adds category-specific pipelines (sofa/chair/bed/table…) for significantly better stability.
Editable Canvas + Grid Adjustments
You can resize and align results in the canvas now. Full version adds grid-based proportional adjustments.
Data Handling
Uploads are used for inference and not stored by default. Only assets you explicitly save to your library are retained.
Built for Integration
Designed to plug into real workflows (presentations, layout planning, catalogs, batch pipelines) — not to replace CAD or full 3D.

FAQ

Is the output true 3D?
No. The output is 2D images rendered in a strict isometric projection, not true 3D geometry.

This is a deliberate choice. By staying in 2D with enforced isometric rules, the system stays much lighter, faster, and easier to integrate into real workflows, while still preserving spatial readability.
Can it generate back or rear views of furniture?
Yes.

The system can generate back views, but this option is currently disabled in the public demo to keep the experience fast and focused. Rear and alternative views will be available in the full version.
Are angles and style consistent? Will results vary each time?
In about 95% of cases, the desired angle can be found within the four generated outputs.

That said, irregular or highly asymmetrical objects are naturally less stable. At the moment, all furniture types share a single core algorithm. In the full release, category-specific algorithms (sofas, chairs, beds, tables, etc.) will be used, which significantly improves consistency and stability.
How well are materials and textures preserved? Will AI blur or "smear" details?
With reasonably clear input images, materials and textures are preserved at full fidelity, including fine details.

The system is rule-constrained to prioritize structural and material accuracy rather than stylistic reinterpretation.
Can I fine-tune size, proportion, or alignment — or am I stuck with the output?
You can already resize and align assets inside the canvas.

In the full version, proportions will also be adjustable using a grid-based system. Proportional editing is not available in the current demo.
Which furniture categories are supported?
Currently supported: Most furniture categories except rugs, wall art, and plants.

These excluded categories will be supported in the full release.
What are the input image requirements? Can I use phone photos?
The demo currently supports PNG and JPEG, up to 4MB per image.

Phone photos are acceptable, but for best results in the demo, they should be cropped and converted to PNG. The full version will support HEIC and original phone photos directly.
Are there copyright or data-usage concerns if I use brand furniture photos?
Uploaded images are used only to help the system understand the object.

They are not stored or added to any training database unless the user explicitly saves them to their personal library.
Do you support APIs or bulk processing for furniture brands?
Yes.

API access and bulk conversion are supported, and pilot partnerships are actively welcome. We are currently working with a home-goods startup in New York. More details are available in the Ecosystem → Shop section.
